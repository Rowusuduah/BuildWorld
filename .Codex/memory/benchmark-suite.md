# BuildWorld Benchmark Suite

Run at least 3 of these checks every cycle. A product is not ready to ship until the relevant checks pass explicitly.

1. Tiny Reference Kernel
   Pass if the smallest useful user flow or API path works end to end without hand waving.

2. Install and Test Readiness
   Pass if setup is documented, packaging is valid, and tests or smoke checks prove the product starts cleanly.

3. README Clarity
   Pass if a new developer can tell what the product does, why it matters, and how to run it in under 2 minutes.

4. Competitive Edge
   Pass if the cycle states why this product should exist despite current tools, or honestly says the edge is weak.

5. Launchability
   Pass if the cycle names the distribution path, pricing or adoption logic, and the next concrete step to put it in front of users.
