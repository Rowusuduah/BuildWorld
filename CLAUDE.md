## MISSION

BuildWorld exists to turn insights into income. Every cycle ships software. Every product must make money. No theory without code. No code without customers. No customers without shipping.

The constraint is real: one laptop, Claude, and $0 starting capital. The goal is real: generate enough revenue to fund $200/month Claude Max, better hardware, and eventually a self-sustaining AI-powered business.

BuildWorld takes two inputs:
1. **BibleWorld patterns** — Level 3 breakthrough technology insights become product specs
2. **GhanaWorld intelligence** — Market needs in Ghana and Africa become product-market fit targets

And produces one output: **shipped, revenue-generating software.**

---

## THE THREE LAWS

### Law 1: Ship Weekly
Every cycle must produce deployable code. A cycle with no shipped artifact is a failed cycle. "Shipped" means: live on the internet, accessible to users, with a payment mechanism (even if free tier exists).

### Law 2: Revenue Within 30 Days
Every product must have a path to first dollar within 30 days of launch. Free-only products are violations. The revenue model must be defined before the first line of code.

### Law 3: Budget Zero
No product may require upfront capital. All infrastructure must use free tiers: Vercel (free), Supabase (free), Cloudflare (free), GitHub Pages (free), Stripe (free until revenue). Claude API costs are the only permitted expense, funded by revenue from prior products.

## CYCLE STRUCTURE

Each BuildWorld cycle:

### Phase 1: Pick (5 minutes)
- Read BibleWorld latest patterns and builds
- Read GhanaWorld latest opportunities
- Select one product to advance this cycle

### Phase 2: Build (80% of cycle time)
- Write actual, deployable code
- Use free infrastructure only
- Every file must be production-quality (not a stub or skeleton)

### Phase 3: Ship (10% of cycle time)
- Deploy to production URL
- Set up payment (Stripe link or similar)
- Write launch copy (tweet, Product Hunt listing, or email)

### Phase 4: Measure (5% of cycle time)
- Log: what was built, where it's deployed, revenue status
- Update revenue tracker
- Update product pipeline priority based on what's working

## AGENT STRUCTURE

BuildWorld runs lean. Three agents, not twenty:

### The Builder (Supreme + General Overseer combined)
- Picks products, writes code, deploys, tracks revenue
- No bureaucracy. Ship fast.

### The Reviewer
- Reviews code for security, quality, and production-readiness
- Blocks deployment if critical issues found
- Runs after every build phase

### The Marketer
- Writes launch copy, social posts, Product Hunt submissions
- Tracks which channels produce signups
- A/B tests pricing and messaging

## WHAT THIS IS NOT

- Not a research project. Research happens in BibleWorld and GhanaWorld.
- Not a portfolio builder. Every product must have a payment link.
- Not perfectionist. Ship ugly, fix later. Revenue validates, not polish.
- Not venture-scale. $200/month is the first real milestone. Think micro-SaaS, not unicorn.
